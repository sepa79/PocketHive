# InfluxDB Support Bundle

This bundle is intended for vendor review of the ClickHouse vs InfluxDB 3 read benchmark.

Contents:

- `executive-summary.md`: short vendor-facing summary with the main observations
- `methodology.md`: dataset, benchmark method, timing semantics, and caveats
- `mail-template.md`: ready-to-edit email body
- `benchmark-report.json`: full machine-readable benchmark results
- `benchmark-report.md`: human-readable report generated by the harness
- `environment.json`: dataset and bundle metadata
- `queries/query-suite.md`: rendered ClickHouse and InfluxDB SQL for every benchmark window/scenario
- `queries/query-suite.json`: same query suite in machine-readable form
- `dashboards/`: Grafana dashboards used as the benchmark surface

Recommended send pattern:

1. Put the short conclusion in the email body.
2. Attach the full bundle as ZIP.
3. Ask InfluxDB to focus on:
   - client-observed latency vs server-side timing divergence
   - long-window read latency
   - query shapes that are slow or planner-sensitive
