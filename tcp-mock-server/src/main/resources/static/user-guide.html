TCP Mock Server User Guide

Getting Started

TCP Mock Server provides enterprise-grade TCP mocking capabilities for testing and simulation.

Quick Setup

# Start the server
docker-compose up tcp-mock-server

# Access UI
http://localhost:8082

Configuration

Configure the server through environment variables or application.yml:

tcp-mock:
  port: 9090
  ssl:
    enabled: false
    keystore: classpath:keystore.p12
    password: changeit
  validation:
    enabled: true
  filtering:
    enabled: true

Message Types

Define message types for different protocols:

ISO-8583 Example

{
  "id": "ISO8583_AUTH",
  "name": "Authorization Request",
  "pattern": "^0100.*",
  "handler": "ISO8583"
}

Request Mappings

Create mappings to define request/response behavior:

{
  "id": "auth-success",
  "pattern": ".*AUTH.*",
  "response": "0110{{ payload.substring(4) }}00",
  "latency": {
    "type": "uniform",
    "min": 10,
    "max": 50
  }
}

Performance Analytics

Monitor performance with real-time dashboards:

  • Throughput Trends - Requests per second over time
  • Latency Distribution - Response time percentiles
  • Error Analysis - Error types and rates
  • Message Type Performance - Per-type metrics

Enterprise Features

Recording Mode

Capture real TCP traffic and auto-generate mappings:

POST /api/enterprise/recording/toggle

Hot Reload

Reload configuration without server restart:

POST /api/enterprise/config/reload

Cluster Coordination

Multi-node deployment with automatic synchronization:

GET /api/enterprise/cluster/status
