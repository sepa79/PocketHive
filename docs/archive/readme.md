# Documentation Archive

This directory contains completed delivery plans, superseded designs, fixed bug reports, and historical verification evidence.

Archived files are not current product contracts. Use living documentation under `docs/`, component READMEs, and `docs/index.md` for current behavior. Read an archived file only when historical rationale or delivery evidence is needed.

Subdirectories group related evidence and design packs, for example:

- `auth-proving-runs/` — deployed auth acceptance evidence,
- `pockethive-plugin/` — historical MCP/IDE POC and implementation material,
- `scenario-editor-plans/` — superseded editor designs,
- `pockethive_scenario_builder_mvp/` — historical scenario-builder MVP pack.

When archiving a plan:

1. set an explicit `implemented / archived`, `fixed / archived`, or `superseded / archived` status;
2. move any genuinely unfinished work into a small file under `docs/todo/`;
3. update active indexes and links;
4. do not maintain a second active copy of the archived document.
